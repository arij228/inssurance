02 — Pipeline

Extract to insight, six stages.

Each stage is a discrete script or model artifact — reproducible, and independently rerunnable.

01

Extract

Pull raw tables from source systems via SQLAlchemy connectors into staging.

02

Cleanse

Resolve FK mismatches, null handling, dedup keys with pandas.

03

Transform

Reshape into star schema — fact and dimension tables.

04

Load

Write to SQL Server warehouse with constraint validation.

05

Model

Train XGBoost regression, classification & forecasting sets.

06

Visualize

Publish DAX measures and dashboards in Power BI.

🔌 Extract Python · SQLAlchemy Connects to policy, claims, and customer source systems and lands raw tables into a staging schema, preserving original keys for auditability.
🧹 Cleanse Python · pandas Normalizes date formats, resolves duplicate policyholder records, and flags rows failing referential integrity checks before they reach the warehouse.
🔄 Transform Python · SQL Reshapes staged data into the star schema: one claims fact table joined to policyholder, coverage, geography, and time dimensions.
💾 Load SQL Server Bulk-loads the transformed tables with foreign-key constraints enforced, catching silent join failures at load time rather than in the report.
🧠 Model XGBoost · scikit-learn Trains and evaluates the three model families directly against the warehouse — regression, classification, and time-series forecasting.
📊 Visualize Power BI · DAX DAX measures compute loss ratios, exposure, and trend lines on top of the schema, published as the three-page executive report.
